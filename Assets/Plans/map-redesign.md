# プロジェクト概要
- ゲームタイトル: Coin Toss Game
- ハイレベルコンセプト: コインを器に投げ入れるアーケードバトルとローグライク要素を組み合わせたゲーム。
- プレイヤー: シングルプレイヤー
- インスピレーション: Slay the Spire / ローグライクデッキビルドゲーム
- ターゲットプラットフォーム: PC (Windows)
- レンダーパイプライン: URP

# ゲームメカニクス
## コアゲームプレイループ
プレイヤーはマップ上のノード（バトル、イベント、ショップなど）を進みます。バトルではマウスホイールでコインを投げ、器に入れてダメージを与えます。戦闘の間、プレイヤーは体力や所持金を管理します。

## 再設計されたマップシーン
MapSceneは「Slay the Spire 2」を参考に、より多様なノードを搭載します：
- **敵**: BattleSceneに移行。
- **エリート**: BattleSceneに移行。通常の「敵」よりステータスが強化されます。
- **ボス**: マップの最後に配置。BattleSceneに移行し、ボス戦が発生します。
- **商人**: StoreSceneに移行（ショップ）。
- **宝箱**: TreasureScene（新規）に移行。パワーアップアイテムを獲得。
- **休憩**: BreakScene（新規）に移行。体力を回復。
- **不明**: EventSceneに移行（ランダムイベント）。

# UI
## 上部バー (Global HUD)
全シーンで共通して表示される上部ヘッダー：
- **左側**: 体力（数値 + バー）、お金（アイコン + 数値）。
- **中央**: 所持アイテム（パワーアップアイテムのアイコンリスト）。
- **右側**: プレイタイム（HH:MM:SS）、履歴（クリックでサイドパネルを表示）、設定（ギアアイコン）。

## 履歴パネル (History Log Panel)
- 履歴ボタンをクリックすると、画面右側からスライドして表示されるパネル。
- 「50ゴールド獲得」「10ダメージを受けた」「アイテム獲得: ラッキーコイン」などのログを垂直リストで表示。

# 主要アセットとコンテキスト
- `RunManager.cs`: プレイタイム、履歴、最大HPを追跡するように更新。新しいシーンへの遷移処理を追加。
- `MapNodeData.cs`: NodeType 列挙型に新しい種類を追加。
- `MapNode.cs`: 新しいノードの種類に応じたシーン遷移ロジック。
- `GlobalHUD.cs`: 上部バーの再設計と状態同期。
- `HistoryPanel.cs`: 履歴ログパネルのロジック。
- `TreasureScene.unity` & `BreakScene.unity`: 新規プロトタイプシーン。

# 実装ステップ
1. **データモデルの拡張**
   - `MapNodeData.NodeType` に `Elite`, `Treasure`, `Rest` を追加。
   - `RunManager` に `MaxHP`, `PlayTime`, `HistoryLog` (文字列リスト) を追加。
   
2. **Global HUD と履歴パネルの作成**
   - `HUDCanvas` を再設計し、上部バーのレイアウト（体力、金、アイテム、時間、履歴ボタン、設定ボタン）を構築。
   - `HistoryPanel`（UIオブジェクト）と制御スクリプトを作成。
   - `RunManager` でプレイタイムのカウントアップと履歴の記録ロジックを実装。

3. **マップロジックとシーン遷移の更新**
   - `RunManager.cs` に `LoadTreasure()` と `LoadBreak()` を追加。
   - `MapNode.cs` を更新し、すべての新しいノードタイプに対応（ボス、エリート、敵はすべてBattleSceneに遷移）。
   - 遷移時に `EnemyManager` 向けの難易度パラメータを `RunManager` 等を介して設定。

4. **新規シーンプロトタイプの作成**
   - **TreasureScene**: 「宝箱を開ける」ボタンを配置。クリックでアイテムを付与し、履歴に記録してマップに戻る。
   - **BreakScene**: 「休憩する」ボタンを配置。HPを最大値の30%回復し、履歴に記録してマップに戻る。

# 検証とテスト
- **上部バー**: HP、お金、時間がリアルタイムで更新されるか確認。
- **履歴**: アイテム取得やダメージ時に履歴パネルに行が追加されるか確認。
- **シーンフロー**: 各ノード（特にボス、エリート、敵）から正しく対応するシーン（BattleScene等）へ遷移することを確認。
