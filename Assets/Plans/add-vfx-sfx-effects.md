# プロジェクト概要
- ゲームタイトル: CoinTossGame
- コンセプト: マウスホイールの「フリック」操作でコインを器へ投げ入れる、物理ベースのコインプッシャーバトルゲーム。
- プレイヤー: シングルプレイヤー
- ターゲットプラットフォーム: Windows スタンドアロン
- レンダーパイプライン: URP

# ゲームメカニクス
## コアゲームループ
- ウェーブ制のサバイバル。
- プレイヤーフェーズ：マウスホイールでコインを器に投げ入れる。
- ダメージフェーズ：器に入ったコインの数に応じて敵にダメージを与える。
- 敵フェーズ：敵が生き残っている場合、プレイヤーに攻撃を行う。
- パワーアップフェーズ：アップグレードを選択する。

## 操作・入力方法
- マウスホイールを前に回してコインを発射。
- スクロール速度によって発射力が決定。

# UI
- バトルUI：タイマー、スコア、現在のフェーズを表示。
- パワーアップUI：アップグレード選択画面。

# 主要アセットとコンテキスト
- `CoinLauncher.cs`: コインの発射ロジックを担当。
- `Coin` プレハブ: 発射される物理オブジェクト。
- `Main Camera`: シーン内のメインカメラ。
- 音声アセット:
    - `Assets/Sound/SE/Shot/`: 発射音（se_Shot[chun].mp3, se_Shot[pyun].mp3, se_Shot[pyu].mp3）
    - `Assets/Sound/SE/Wall Reflection/`: 壁反射音（se_Coinreflection.wav, se_Coinreflection2.wav）

# 実装ステップ

## 1. カメラシェイクシステムの作成
- **ファイル**: `Assets/Scripts/Gameplay/CameraShake.cs` (新規)
- **内容**: 指定した強度と時間でカメラを揺らすシンプルなコンポーネントを実装。
- **依存関係**: なし

## 2. コイン衝突フィードバックシステムの作成
- **ファイル**: `Assets/Scripts/Gameplay/CoinCollisionFeedback.cs` (新規)
- **内容**: コインのプレハブにアタッチし、名前が「Wall」を含むオブジェクトと衝突した際に反射音を再生する。
- **設定項目**:
    - `AudioClip[] reflectionSounds` (反射音リスト)
    - `float minPitch`, `float maxPitch` (ピッチのランダム範囲)
- **依存関係**: なし

## 3. コインランチャーの拡張（音声・シェイク対応）
- **ファイル**: `Assets/Scripts/Gameplay/CoinLauncher.cs` (修正)
- **内容**: 
    - `AudioSource`、`AudioClip[] shotSounds`、`CameraShake` の参照、および強度・ピッチの範囲を設定するフィールドを追加。
    - `LaunchCoin` メソッドを以下のように修正：
        - ランダムな発射音を再生。
        - 発射力に応じてピッチを変更（力が大きいほど高いピッチ）。
        - 発射力に応じてカメラシェイクをトリガー。
- **依存関係**: ステップ 1

## 4. シーンおよびプレハブのセットアップ
- **プレハブ**: `Assets/Prefabs/Coin.prefab`
    - `AudioSource` コンポーネントを追加。
    - `CoinCollisionFeedback` スクリプトを追加し、反射音を設定。
- **シーン**: `BattleScene`
    - `Main Camera` オブジェクトに `CameraShake` をアタッチ。
    - `CoinLauncher` オブジェクトの設定：
        - `AudioSource` コンポーネントを追加。
        - `CameraShake` への参照を設定。
        - `Assets/Sound/SE/Shot` から発射音をアサイン。
        - ピッチとシェイク強度の範囲を設定。
- **依存関係**: ステップ 1, 2, 3

# 検証およびテスト
- **発射音**: コイン発射時に音が鳴り、ホイールの回転速度によってピッチが変わることを確認。
- **壁反射音**: コインが壁（Wall）に跳ね返った際に音が鳴ることを確認。
- **カメラシェイク**: 発射時にカメラが揺れ、速度が速いほど揺れが強くなることを確認。
- **インスペクター設定**: Unity エディタ上で各パラメータを調整可能であることを確認。
