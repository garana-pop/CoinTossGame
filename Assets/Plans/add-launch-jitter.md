# プロジェクト概要
- ゲームタイトル: CoinTossGame
- ハイレベルコンセプト: マウスホイールの操作でコインを器の中に投げ入れ、その数に応じて敵にダメージを与える物理ベースのバトルゲーム。
- プレイヤー: シングルプレイヤー
- インスピレーション / 参照ゲーム: メダルゲーム、ローグライクライトな進行要素
- トーン / アートディレクション: 物理的・触覚的なフィードバックを重視
- ターゲットプラットフォーム: PC (Windows)
- レンダリングパイプライン: PC_RPAsset (URP またはカスタム)

# ゲームメカニクス
## コアゲームプレイループ
- 制限時間内にコインを器に投げ入れる。
- 物理演算（軌道、速度）が成功の鍵となる。
- 器の中にあるコインの数によってダメージを計算。
- 敵からの反撃。
- ウェーブ間にパワーアップを選択。

# UI
- HP、タイマー、スコアを追跡する最小限のバトルUI。

# 主要アセットとコンテキスト
- `CoinLauncher.cs`: コインの生成と物理演算の適用を担当。
- `GameConstants.cs`: グローバルなゲームバランスの定数を保持。

# 実装ステップ
1. **`CoinLauncher.cs` の修正**:
    - `[SerializeField, Range(0f, 0.5f)] private float jitterAmount = 0.05f;` フィールドを追加。インスペクターで調整可能にする。
    - `LaunchCoin(float force)` メソッドを更新し、発射ベクトルにランダムな揺れ（ジッター）を加える処理を実装。
    - `Random.insideUnitSphere` に `jitterAmount` を乗じたものを前方向ベクトルに加え、再度正規化することで、勢いを保ちつつ方向だけをバラつかせる。
2. **Unityエディタでの確認**:
    - `BattleScene` 内の `CoinLauncher` のインスペクターにスライダーが表示されているか確認。
    - コインを発射し、軌道にランダムな変化が生じているか目視で確認。

# 検証とテスト
- **目視確認**: `jitterAmount` を 0 に設定して複数枚発射し、すべてのコインが全く同じ軌道を通ることを確認。
- **目視確認**: `jitterAmount` を高い値（例: 0.5）に設定し、発射されるコインが大きく散らばることを確認。
- **エッジケース**: `jitterAmount = 0` の場合にエラーが発生せず、従来通りの挙動になることを確認。
- **エッジケース**: 揺れを加えても、発射の勢い（力の大きさ）が意図した通りであることを確認。
